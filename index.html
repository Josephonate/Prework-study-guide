HTML

  • Hypertext Markup Language (HTML)
  • The following element types are commonly used to add content to pages: Text elements Image elements Header elements List elements
  • Here are some common element types that are used to divide and describe a page: Meta-related elements Content division elements Semantic elements
  • The <head> element contains information, or metadata, that helps the browser to render the page correctly. Elements included within the <head> are not visible when a user looks at a webpage.
  • The <body> element contains the content that is shown to the user. All the elements that we want the user to see and interact with must be placed inside this element.
  • The <head> element contains information about the webpage.
  • The <body> element represents the visible content shown to the user.
  • The <title> element defines the title for a webpage. This element is required and is extremely important for Search Engine Optimization (SEO).
  • The <style> element defines the CSS styles associated with a website.
  • The <base> element defines the base URL for a webpage.
  • The <link> element connects an external resource to the HTML document.
  • The <meta> element defines metadata such as the character set, description, keywords, author, and viewport.
  • The <script> element embeds client-side scripts in the HTML document.

CSS

  • To add style to our webpage, we will use Cascading Style Sheets (CSS)
  • CSS defines the colors, aesthetics, visual layout, and organization.
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • When a developer wants to style an element directly in the HTML file, they use inline CSS
  • useful for quick and permanent changes but are less flexible than external and internal style sheets
  • This internal style sheet contains CSS rules for the webpage in the head section of the HTML file.
  • the most common way to style a webpage is by using an external CSS style sheet.
  • his is preferred in most cases because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A stages your changes
  • git commit -m "<your message here>" describes the changes you made
  • git pull origin Make sure the local branch is up to date before pushing up the changes. Pull in the base branch main to your feature branch.
  • git push origin push changes to the remote repository.
